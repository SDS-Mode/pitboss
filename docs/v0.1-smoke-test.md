# Agent Shire v0.1 — Initial-Run Smoke Test Procedure

**Purpose:** Verify the v0.1 implementation works end-to-end before promoting `v0.1.0-pre` → `v0.1.0`.
Exercises behaviors the automated test suite can't fully cover: real `claude` subprocess, real
stream-json output, real signal handling, real artifacts on disk.

**Estimated runtime:** 20–40 minutes of wall-clock, most of it waiting on real Claude API calls.
**Estimated cost:** ~$0.30–$1.00 in Anthropic API usage depending on which model is used.

---

## Prerequisites

- [ ] `shire` binary built and on `$PATH` (or note its absolute path).
  ```bash
  cd /run/media/system/Dos/Projects/agentshire
  cargo build --release -p shire-cli
  export PATH="$(pwd)/target/release:$PATH"
  shire version   # expect: "shire 0.1.0"
  ```
- [ ] `claude` CLI installed and already authenticated via its existing subscription
  configuration (OAuth / Claude.ai login — no `ANTHROPIC_API_KEY` needed on this host).
  Test: `claude --version` prints a version. For a deeper check that auth is live,
  run `echo 'reply with just the word ready' | claude -p --output-format stream-json --verbose`
  once by hand and confirm you get stream-json output without an auth error.
- [ ] A clean scratch directory for the test run:
  ```bash
  export SHIRE_SMOKE_DIR="$HOME/shire-smoke-test"
  rm -rf "$SHIRE_SMOKE_DIR" && mkdir -p "$SHIRE_SMOKE_DIR"
  cd "$SHIRE_SMOKE_DIR"
  ```
- [ ] `git` ≥ 2.5 (for worktree support).
- [ ] `jq` for summary.json inspection (optional but convenient).
- [ ] No `ANTHROPIC_MAX_CONCURRENT` set in env (we want to control precedence explicitly):
  ```bash
  unset ANTHROPIC_MAX_CONCURRENT
  ```

## Part 1 — Offline tests (no API calls)

These tests don't invoke `claude` at all; they validate CLI plumbing, manifest handling, and
exit-code semantics. Each should complete in <1s.

### 1.1 Version

```bash
shire version
```
**Expect:** exactly `shire 0.1.0`. Exit 0.
- [ ] Pass

### 1.2 Help

```bash
shire --help
shire dispatch --help
shire validate --help
```
**Expect:** readable help text for each. Exit 0.
- [ ] Pass

### 1.3 Validate — happy path

```bash
# Setup a throwaway git repo
mkdir -p "$SHIRE_SMOKE_DIR/repo-a"
cd "$SHIRE_SMOKE_DIR/repo-a"
git init -q && git config user.email t@t.x && git config user.name t
echo "hi" > README.md && git add . && git commit -q -m "init"
cd "$SHIRE_SMOKE_DIR"

cat > happy.toml <<EOF
[[task]]
id = "smoke"
directory = "$SHIRE_SMOKE_DIR/repo-a"
prompt = "hello"
EOF

shire validate happy.toml
```
**Expect:** `OK — 1 tasks, max_parallel=4`. Exit 0.
- [ ] Pass

### 1.4 Validate — unknown-field rejection

```bash
cat > bad-key.toml <<EOF
bogus_field = "surprise"
[[task]]
id = "x"
directory = "$SHIRE_SMOKE_DIR/repo-a"
prompt = "p"
EOF

shire dispatch bad-key.toml
echo "exit=$?"
```
**Expect:** stderr mentions `unknown field` or similar parse error. Exit code 2.
- [ ] Pass

### 1.5 Validate — missing directory

```bash
cat > bad-dir.toml <<EOF
[[task]]
id = "x"

shire dispatch bad-dir.toml

```
**Expect:** stderr mentions directory does not exist. Exit 2.
- [ ] Pass

### 1.7 Validate — duplicate task ids

```bash
cat > bad-dups.toml <<EOF
[[task]]
id = "same"
```

### 1.8 Dry-run

```bash
shire dispatch --dry-run happy.toml
echo "exit=$?"
```
**Expect:** prints one `DRY-RUN smoke:` line showing the claude command + args (should include
`--output-format stream-json`, `-p hello`, and the path to the `claude` binary). Exit 0.
No files written to `~/.local/share/shire/runs/`.
- [ ] Pass

### 1.9 Missing claude binary

```bash

```
**Expect:** stderr mentions "claude binary not found". Exit 2.
- [ ] Pass

### 1.10 ANTHROPIC_MAX_CONCURRENT precedence

```bash
cat > concurrency.toml <<EOF
[[task]]
id = "a"
directory = "$SHIRE_SMOKE_DIR/repo-a"
prompt = "p"
EOF

shire validate concurrency.toml                                     # expect max_parallel=4
ANTHROPIC_MAX_CONCURRENT=7 shire validate concurrency.toml          # expect max_parallel=7

cat > concurrency-manifest.toml <<EOF
[run]
max_parallel = 2
[[task]]
id = "a"
directory = "$SHIRE_SMOKE_DIR/repo-a"
prompt = "p"
EOF

ANTHROPIC_MAX_CONCURRENT=7 shire validate concurrency-manifest.toml # expect max_parallel=2
```
**Expect:** three different `max_parallel` values in output (4, 7, 2) confirming precedence
manifest > env > default.
- [ ] Pass

## Part 2 — Online tests (real `claude` binary + API)

Each of these spawns real subprocesses that call the Anthropic API. Keep prompts short and
cheap. Use `--model claude-haiku-4-5` (or cheapest available) unless a test specifically
needs a different model.

**Set a common env for all Part 2 tests:**
```bash
cd "$SHIRE_SMOKE_DIR"
# Ensure claude is resolvable
which claude
```

### 2.1 Single-task happy path

```bash
cat > t1-happy.toml <<EOF
[run]
worktree_cleanup = "always"

[defaults]
model = "claude-haiku-4-5"
timeout_secs = 120
use_worktree = true

[[task]]
id = "write-hi"
directory = "$SHIRE_SMOKE_DIR/repo-a"
prompt = "Create a file named hi.txt containing the single word hello."
branch = "smoke/write-hi"
EOF

shire dispatch t1-happy.toml
echo "exit=$?"
```

**Expect during run:**
- Progress table updates in place (if TTY): `write-hi` goes from `… Pending` → `● Running` → `✓ Success`.
- Elapsed time shown in `Xm YYs` format.
- Tokens shown once result event arrives.
- Exit code 0.

**Expect after run:**
```bash
# Find the run directory
RUN_DIR=$(ls -td ~/.local/share/shire/runs/*/ | head -1)
echo "run: $RUN_DIR"
ls -la "$RUN_DIR"
```
- [ ] `manifest.snapshot.toml` present and matches the input manifest byte-for-byte.
- [ ] `resolved.json` present and parseable as JSON containing `tasks[0].id = "write-hi"`.
- [ ] `meta.json` present, contains `shire_version: "0.1.0"` and a `claude_version` string (or null).
- [ ] `summary.jsonl` present with exactly one line (the single task record).
- [ ] `summary.json` present, `tasks_total: 1`, `tasks_failed: 0`, `was_interrupted: false`.
- [ ] `tasks/write-hi/stdout.log` present, non-empty, contains stream-json lines.
- [ ] The branch `smoke/write-hi` exists in the repo (`git -C "$SHIRE_SMOKE_DIR/repo-a" branch --list smoke/write-hi`).
- [ ] `hi.txt` exists in the checkout of `smoke/write-hi` (`git -C "$SHIRE_SMOKE_DIR/repo-a" show smoke/write-hi:hi.txt`).
- [ ] Worktree sibling directory was cleaned up (no `repo-a-shire-*` dirs next to repo-a).

### 2.2 Token usage is non-zero and reasonable

```bash
jq '.tasks[0].token_usage' "$RUN_DIR/summary.json"
```
**Expect:** `input`, `output`, `cache_read`, `cache_creation` all ≥ 0, with `input > 0` and
`output > 0`. Token counts should be in the hundreds-to-thousands range for a simple prompt.
- [ ] Pass

### 2.3 `final_message_preview` is truncated/reasonable

```bash
jq '.tasks[0].final_message_preview' "$RUN_DIR/summary.json"
```
**Expect:** a short string (≤ 201 chars), ending with `…` if truncated from a longer message.
Should be human-readable text that looks like Claude's last assistant message before completion.
- [ ] Pass

### 2.4 `claude_session_id` present

```bash
jq '.tasks[0].claude_session_id' "$RUN_DIR/summary.json"
```
**Expect:** non-null string that looks like a session id. This confirms the parser successfully
extracted the session id from the `result` event and can later be used for `--resume` (future milestone).
- [ ] Pass

### 2.5 Two-task parallel dispatch

```bash
# Second repo for true isolation
mkdir -p "$SHIRE_SMOKE_DIR/repo-b"
cd "$SHIRE_SMOKE_DIR/repo-b" && git init -q && git config user.email t@t.x && git config user.name t
echo "hi" > README.md && git add . && git commit -q -m "init"
cd "$SHIRE_SMOKE_DIR"

cat > t2-parallel.toml <<EOF
[run]
max_parallel = 2
worktree_cleanup = "always"

[defaults]
model = "claude-haiku-4-5"
timeout_secs = 120

[[task]]
id = "a-task"
directory = "$SHIRE_SMOKE_DIR/repo-a"
prompt = "Create file a.txt with content A."
branch = "smoke/a"

[[task]]
id = "b-task"
directory = "$SHIRE_SMOKE_DIR/repo-b"
prompt = "Create file b.txt with content B."
branch = "smoke/b"
EOF

time shire dispatch t2-parallel.toml
echo "exit=$?"
```
**Expect:**
- Both tasks progress concurrently (observe progress table).
- Wall-clock time is roughly max(t_a, t_b), not t_a + t_b.
- Exit 0.
- `summary.json` shows `tasks_total: 2`, `tasks_failed: 0`.
- Both files (`a.txt`, `b.txt`) created on respective branches.
- [ ] Pass

### 2.6 Template expansion

```bash
cat > t3-template.toml <<EOF
[defaults]
model = "claude-haiku-4-5"
use_worktree = false

[[template]]
id = "greet"
prompt = "Write the word {greeting} followed by {name} to stdout, then finish."

[[task]]
id = "english"
directory = "$SHIRE_SMOKE_DIR/repo-a"
template = "greet"
vars = { greeting = "hello", name = "world" }
EOF

shire dispatch t3-template.toml
echo "exit=$?"

RUN_DIR=$(ls -td ~/.local/share/shire/runs/*/ | head -1)
jq '.tasks[0].final_message_preview' "$RUN_DIR/summary.json"
```
**Expect:** `final_message_preview` mentions "hello" and "world" (confirms the template
was substituted before being handed to the subprocess). Exit 0.
- [ ] Pass

### 2.7 halt_on_failure cascade

```bash
cat > t4-halt.toml <<EOF
[run]
halt_on_failure = true
max_parallel = 1

[defaults]
model = "claude-haiku-4-5"
timeout_secs = 60
use_worktree = false

[[task]]
id = "will-fail"
directory = "$SHIRE_SMOKE_DIR/repo-a"
prompt = "Immediately exit with error. Do not do anything. Just fail."

[[task]]
id = "should-be-skipped"
directory = "$SHIRE_SMOKE_DIR/repo-a"
prompt = "Create skip.txt containing SKIPPED."
EOF

shire dispatch t4-halt.toml
echo "exit=$?"
```

**Note:** We can't easily force claude to exit non-zero on purpose; this test is
best-effort. If the first task actually succeeds, move on — the automated integration
test `halt_on_failure_stops_remaining_tasks` already exercises the cascade logic deterministically.

If the first task DOES fail:
- Exit code 1.
- `summary.json` `tasks` array has length 1 (only "will-fail" recorded; "should-be-skipped" never ran).
- [ ] Pass  OR  [ ] Skipped (first task unexpectedly succeeded)

### 2.8 Timeout enforcement

```bash
cat > t5-timeout.toml <<EOF
[defaults]
model = "claude-haiku-4-5"
use_worktree = false

[[task]]
id = "slow"
directory = "$SHIRE_SMOKE_DIR/repo-a"
timeout_secs = 5
prompt = "Count from 1 to 100000 out loud, one number per line, no shortcuts."
EOF

time shire dispatch t5-timeout.toml
echo "exit=$?"

RUN_DIR=$(ls -td ~/.local/share/shire/runs/*/ | head -1)
jq '.tasks[0].status' "$RUN_DIR/summary.json"
```
**Expect:**
- Dispatch returns in roughly `timeout_secs + TERMINATE_GRACE (10s)` = ~15 seconds.
- `status` is `"TimedOut"`.
- Exit 1 (a non-Success task).
- [ ] Pass

### 2.9 Ctrl-C — first signal drains, second terminates

Manual, requires interactive terminal.

```bash
cat > t6-hold.toml <<EOF
[defaults]
model = "claude-haiku-4-5"
use_worktree = false
timeout_secs = 600

[[task]]
id = "long-running"
directory = "$SHIRE_SMOKE_DIR/repo-a"
prompt = "Count very carefully from 1 to 100 with detailed explanations of each number."
EOF

# Interactive run:
shire dispatch t6-hold.toml
```

After the progress table shows `● Running`, **press Ctrl-C once**.

**Expect (first Ctrl-C):** stderr emits `received Ctrl-C — draining; send another within 5s to terminate`. The task continues running for up to `timeout_secs`.
- [ ] Drain message observed

Within 5 seconds, **press Ctrl-C again**.

**Expect (second Ctrl-C):** stderr emits `received second Ctrl-C — terminating subprocesses`.
Task is SIGTERM'd. After up to `TERMINATE_GRACE` (10s), SIGKILL'd if still alive. Dispatch
exits with code 130.
- [ ] Second message observed
- [ ] Exit code 130 (`echo $?` after the run)
- [ ] `summary.json` shows `tasks[0].status == "Cancelled"` and `was_interrupted: true`

### 2.10 Worktree cleanup policies

Three invocations, one per policy.

```bash
# on_success (default)
cat > t7-wt-on-success.toml <<EOF

worktree_cleanup = "on_success"

[defaults]
model = "claude-haiku-4-5"
timeout_secs = 60

[[task]]

directory = "$SHIRE_SMOKE_DIR/repo-a"
prompt = "Create x1.txt with content ONE."
branch = "smoke/wt-on-success"
EOF
shire dispatch t7-wt-on-success.toml
```
- [ ] Worktree sibling dir is **gone** after success (check: `ls "$SHIRE_SMOKE_DIR"/repo-a-shire-*` should be empty).

```bash
# never
cat > t7-wt-never.toml <<EOF
[run]
worktree_cleanup = "never"

[defaults]

[[task]]
```

- [ ] `git -C "$SHIRE_SMOKE_DIR/repo-a" worktree list` shows the persisted worktree.

## Part 3 — Artifact schema audit

Pick one successful run from Part 2 (e.g., 2.1 or 2.5) and walk the artifact schema.

```bash
RUN_DIR=$(ls -td ~/.local/share/shire/runs/*/ | head -1)
cd "$RUN_DIR"
ls -la
```

- [ ] Files present: `manifest.snapshot.toml`, `resolved.json`, `meta.json`, `summary.jsonl`, `summary.json`.
- [ ] Directory present: `tasks/<task-id>/`.
- [ ] Per-task files: `stdout.log` (non-empty), optional `stderr.log`.

```bash
jq . summary.json | head -40
```

Confirm these top-level fields exist on `summary.json`:
- [ ] `run_id` (UUIDv7)
- [ ] `manifest_path` (absolute path)
- [ ] `shire_version` = `"0.1.0"`
- [ ] `claude_version` (string or null)
- [ ] `started_at`, `ended_at` (ISO 8601)
- [ ] `total_duration_ms` (integer, ≥ 0)
- [ ] `tasks_total`, `tasks_failed` (integers)
- [ ] `was_interrupted` (bool)
- [ ] `tasks` (array of TaskRecord)

Per-task (`tasks[]`):
- [ ] `task_id`, `status`, `exit_code`
- [ ] `started_at`, `ended_at`, `duration_ms`
- [ ] `worktree_path` (set when use_worktree=true)
- [ ] `log_path` (points to the stream-json log)
- [ ] `token_usage` with all four fields
- [ ] `claude_session_id` (or null)
- [ ] `final_message_preview` (or null)
