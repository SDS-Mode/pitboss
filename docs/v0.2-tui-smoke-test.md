# Mosaic TUI v0.2-alpha — Smoke Test

Walk-through for verifying the TUI works end-to-end. Takes ~5 minutes. No API
calls; relies on existing run artifacts in `~/.local/share/shire/runs/`.

## Prerequisites

```bash
cd /run/media/system/Dos/Projects/agentshire
cargo build --release -p mosaic-tui
MOSAIC="$(pwd)/target/release/mosaic"
```

Confirm `~/.local/share/shire/runs/` has at least one run. If not, create one:

```bash
SCRATCH=$(mktemp -d)
cd "$SCRATCH" && git init -q && git config user.email t@t.x && git config user.name t
echo x > README.md && git add . && git commit -q -m i
cat > shire.toml <<EOF
[defaults]
model = "claude-haiku-4-5"
timeout_secs = 60

[[task]]
id = "smoke"
directory = "$SCRATCH"
prompt = "write the word ready to a file named ok.txt"
branch = "smoke/check"
EOF
shire dispatch shire.toml
```

## 1 — `mosaic --help`

```bash
$MOSAIC --help
```

**Expect:** prints a help block listing `list` subcommand and the version. Exit 0.
- [ ] Pass

## 2 — `mosaic --version`

```bash
$MOSAIC --version
```

**Expect:** prints `mosaic 0.1.0` (version matches workspace).
- [ ] Pass

## 3 — `mosaic list`

```bash
$MOSAIC list
```

**Expect:** tabular output like

```
RUN ID                                  STARTED                  TASKS  FAILED  STATUS
────────────────────────────────────────────────────────────────────────────────
019d99... 2026-04-17 02:36:40 UTC       1       0  complete
...
```

Sorted newest-first. Exit 0 even if no runs exist (prints a helpful message instead).
- [ ] Pass

## 4 — No-runs message

```bash
# Temporarily point HOME at an empty dir
HOME=/tmp $MOSAIC list
```

**Expect:** `No runs directory found at /tmp/.local/share/shire/runs/.` plus a
`Run `shire` first to create a run.` follow-up. Exit 0.
- [ ] Pass

## 5 — Open most recent run (INTERACTIVE)

```bash
$MOSAIC
```

**Expect:** TUI launches. You should see:
- A top status bar with the run id, N/total task progress, token counts, duration
- A grid of tiles — one per task — with status icons
- A focus pane below showing parsed stream-json events (lines starting with `>`, `*`, `<`, or `v`)
- A bottom keybindings hint bar

**Checks:**
- [ ] Top bar shows something like `Mosaic — run 019d99 — 1/1 done, 0 failed — 18/504 tokens — ...`
- [ ] At least one tile rendered with border, task id, status icon, duration
- [ ] Focus pane shows at least one line of parsed output (not raw JSON)
- [ ] Press `h` or `←` — focus moves (if multiple tiles; otherwise no-op)
- [ ] Press `?` — help overlay appears
- [ ] Press `Esc` — help overlay closes
- [ ] Press `L` — full log overlay appears
- [ ] Press `L` again — log overlay closes
- [ ] Press `q` — TUI exits cleanly, terminal returns to normal

## 6 — Open specific run by prefix

```bash
# Get a run id from `mosaic list`
SOME_RUN=$($MOSAIC list | awk 'NR==3 {print substr($1,1,8)}')
echo "opening: $SOME_RUN"
$MOSAIC "$SOME_RUN"
```

**Expect:** TUI opens that specific run (prefix-matches the full UUID). Top bar
shows the matched run id. Press `q` to exit.
- [ ] Pass

## 8 — Live updates while shire runs (INTERACTIVE, ~1 min)

In one terminal:
```bash
SCRATCH=$(mktemp -d)
cd "$SCRATCH" && git init -q && git config user.email t@t.x && git config user.name t
echo x > README.md && git add . && git commit -q -m i
cat > shire.toml <<EOF
[run]
max_parallel = 2

[defaults]
model = "claude-haiku-4-5"
timeout_secs = 120

[[task]]
id = "t-alpha"
directory = "$SCRATCH"
prompt = "say the word alpha"
branch = "smoke/alpha"

[[task]]
id = "t-beta"
directory = "$SCRATCH"
prompt = "say the word beta"
branch = "smoke/beta"
EOF
shire dispatch shire.toml
```

In another terminal, within ~2 seconds:
```bash
$MOSAIC
```

**Expect:**
- Tiles `t-alpha` and `t-beta` both appear.
- Statuses progress: Pending → Running (cyan `●`) → Done (green `✓`).
- Duration counters tick up during Running phase.
- Top bar's done count increments as tasks complete.
- Focus pane shows live parsed events for the focused tile.

When both tiles reach Done, the shire command in the first terminal also exits.

- [ ] Live updates visible
- [ ] Status transitions observed
- [ ] Focus pane shows events live
- [ ] TUI exits cleanly with `q` after shire finishes

## 9 — Non-TTY behavior

```bash
echo "" | $MOSAIC 2>&1 | head -2
echo "exit=$?"
```

**Expect:** an error mentioning the terminal (crossterm can't enable raw mode
without a TTY). Non-zero exit. `list` works in non-TTY contexts; the TUI does
not and fails explicitly rather than hanging.
- [ ] Pass
